JSX

JSX，是一个 JavaScript 的语法扩展。建议在 React 中配合使用 JSX，JSX 可以很好地描述 UI 应该呈现出它应有交互的本质形式。JSX 可能会使人联想到模版语言，但它具有 JavaScript 的全部功能。

使用 JSX

最终编译

React 认为渲染逻辑本质上与其他 UI 逻辑内在耦合，比如，在 UI 中需要绑定处理事件、在某些时刻状态发生变化时需要通知到 UI，以及需要在 UI 中展示准备好的数据。
React 并没有采用将标记与逻辑进行分离到不同文件这种人为地分离方式，而是通过将二者共同存放在称之为“组件”的松散耦合单元之中，来实现关注点分离。
在 JSX 语法中，你可以在大括号内放置任何有效的 JavaScript 表达式

JSX 特定属性
在属性中嵌入 JavaScript 表达式时，不要在大括号外面加上引号。你应该仅使用引号（对于字符串值）或大括号（对于表达式）中的一个，对于同一属性不能同时使用这两种符号。
JSX 遵循小驼峰命名（camelCase），如 class 样式写成 className tabindex 写成 tabIndex

元素渲染

元素是构成 React 应用的最小砖块。
与浏览器的 DOM 元素不同，React 元素是创建开销极小的普通对象。React DOM 会负责更新 DOM 来与 React 元素保持一致
使用 React 构建的应用通常只有单一的根 DOM 节点。
react 元素是不可变对象。一旦被创建，你就无法更改它的子元素或者属性。一个元素就像电影的单帧：它代表了某个特定时刻的 UI。
改变 react 元素后，可以使用 ReactDOM.render()更新 UI 但大多数 React 应用只会调用一次 ReactDOM.render()，因此需要设置组件的状态来更新 UI

注：
diff 算法：React DOM 会将元素和它的子元素与它们之前的状态进行比较，并只会进行必要的更新来使 DOM 达到预期的状态。

数据是向下流动的

这通常会被叫做“自上而下”或是“单向”的数据流。任何的 state 总是所属于特定的组件，而且从该 state 派生的任何数据或 UI 只能影响树中“低于”它们的组件。

总结：

  • class 组件应该始终使用 props 参数来调用父类的构造函数。
  • 挂载在组件的功能，要在组件卸载时及时卸载（防止内存泄漏）
  • 不要直接修改 state，state 只能在 class 的构造函数中被赋初值，其他地方直接修改 state 不会有响应
  • 更新状态需要使用 setState 方法，setState 会触发组件的 render 方法，从而引起 UI 重绘
  • 组件的 state、props 的更新可能是异步的，不要依赖他们的值来更新下一个状态。
  • 可以使用 setState 单独更新要更新的属性，state 的更新会被合并
  • 数据是向下流动的（单向数据流），从该 state 派生的任何数据或 UI 只能影响树中“低于”它们的组件。
  • 任何的 state 总是所属于特定的组件，具有封装性和局部性
  • 可以向 class 组件中随意添加不参与数据流（比如计时器 ID）的额外字段。

列表循环

循环的每一个元素建议添加 key 属性用于做唯一标识，key 值可以绑定 id 等，没有 id 也可以绑定 index

key 帮助 React 识别哪些元素改变了，比如被添加或删除。因此你应当给数组中的每一个元素赋予一个确定的标识。

如果列表项目的顺序可能会变化，我们不建议使用索引来用作 key 值，因为这样做会导致性能变差，还可能引起组件状态的问题。如果你选择不指定显式的 key 值，那么 React 将默认使用索引用作为列表项目的 key 值。

片段

类似vue的template标签和小程序的block标签，这类标签一个常见模式是为一个组件返回多个元素。 片段(fragments) 可以让你将子元素列表添加到一个分组中，并且不会在DOM 中增加额外节点。

简写语法：

可以像使用其他元素一样使用<></>，只是它不支持 键(keys) 或 属性(attributes)。

请注意， 目前许多工具都不支持这个简写语法 ， 所以你可能需要明确地使用 <React.Fragment> ，直到工具支持这个语法。
